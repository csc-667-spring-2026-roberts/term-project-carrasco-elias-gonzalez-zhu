### Creating Migration Files

Create the following migrations:

```bash
npm run migrate:create -- create-games-table
npm run migrate:create -- create-game-users-table
```

This will generate files in the `migrations/` folder similar to:

```text
migrations/
├─ 1234567891234_create-games-table.ts
└─ 1234567891234_create-game-users-table.ts
```

👉 The numeric prefix is a timestamp automatically generated by `node-pg-migrate`.  
👉 Your filenames should match this pattern.

### Implementation Notes

- Implement the tables inside the generated migration files
- Follow the structure from:
  - docs/examples/_create-games-table.example.ts
  - docs/examples/_create-game-users-table.example.ts

👉 Do NOT modify the example files.  
👉 Your migration files must match the expected schema exactly.

### Database Schema Requirements

#### games table

Must include:

- id (primary key)
- status (VARCHAR(20), NOT NULL, default "waiting")
- created_at (timestamp, default now)

Constraints:

- status must be limited to:
  - waiting
  - in_progress
  - finished

#### game_users table

Must include:

- game_id (foreign key → games.id)
- user_id (foreign key → users.id)
- joined_at (timestamp, default now)

Constraints:

- composite primary key on (game_id, user_id)
- cascade delete when game is deleted

### DB Function Requirements (src/db/games.ts)

You must implement:

- createGame(userId)
- listGames()

#### createGame

- insert a new row into games
- insert the creator into game_users
- use default values where appropriate

#### listGames

- return all games for the lobby
- include relevant display data
- order by newest first

## Scripts

Use the following scripts:

- `npm run setup` — Install project dependencies using the shared repo setup script
- `npm run setup:clean` — Remove `node_modules` and reinstall dependencies from scratch

- `npm run migrate:create -- <name>` — Create a new migration file in the migrations/ directory
- `npm run migrate:up` — Apply all pending migrations to the database
- `npm run migrate:down` — Roll back the most recent migration

- `npm run db:recreate` — Drop and recreate the local database (clean reset)

- `npm run db:clear:check` — Clear tables (users, session, games, game_users) and verify they are empty

- `npm run build` — Compile TypeScript
- `npm run lint` — Check for lint errors
- `npm run format:check` — Verify code formatting (Prettier)
- `npm run format` — Automatically format code using Prettier

### Script Guidelines

You may create temporary helper scripts in your local branch if needed to support development or testing.

- You may add new scripts inside the `"scripts"` section of `package.json` for your own workflow
- Do NOT modify or remove existing shared scripts without team agreement
- Do NOT rename existing scripts
- These scripts are optional and for local use only
- Do NOT rely on custom scripts for required validation

👉 Required validation must still be done using the standard scripts listed above.

After implementing your migration files (`npm run migrate:create -- <name>`), apply them:

#### When you need to fix a migration

If a migration is incorrect:

```bash

```

Then:

- update the migration file
- re-run:

```bash
npm run migrate:up
```

Verify the fix worked.

#### When testing database behavior

Use:

```bash

```

This will:

- clear tables (users, session, games, game_users)
- confirm the database is empty before testing

#### If your database gets into a bad state

Reset everything:

```bash

```

Then re-test your changes.

#### When checking your code

```bash

```

Fix formatting:

```bash

```

#### When verifying schema in psql

After running:

```bash
npm run migrate:up
```

Open psql (Terminal 2):

```bash
npx dotenv -e .env -- psql "$DATABASE_URL"
```

#### Check tables exist

```sql

```

Verify:

- games table exists
- game_users table exists

#### Inspect games table

```sql
\d games
```

Verify:

- id, status, created_at columns exist
- status has default value 'waiting'
- status check constraint exists

#### Inspect game_users table

```sql
\d game_users
```

Verify:

- game_id, user_id, joined_at columns exist
- composite primary key exists
- foreign keys reference games.id and users.id
- ON DELETE CASCADE is present

#### Exit psql

```sql
\q
```

#### General workflow

```text
1. implement migration
2. npm run migrate:up
3. verify schema in psql
4. test inserts and constraints
5. if incorrect → npm run migrate:down
6. fix migration
7. repeat
```

## Local Testing Checklist

Before moving forward, verify the database manually.

### 1. Apply migrations (Terminal 1)

Open a terminal and run:

```bash
npm run migrate:up
```

Verify:

- migrations run without errors
- no warnings or failures appear

### 2. Open psql (Terminal 1)

In the same terminal, run:

```bash
npx dotenv -e .env -- psql "$DATABASE_URL"
```

### 3. Confirm the tables exist

Inside psql, run:

```sql
\dt
```

Verify that both of these tables exist:

- games
- game_users

### 4. Inspect the games table

Inside psql, run:

```sql
\d games
```

Verify:

- id exists
- status exists
- created_at exists
- status has default value 'waiting'
- status check constraint exists

### 5. Inspect the game_users table

Inside psql, run:

```sql
\d game_users
```

Verify:

- game_id exists
- user_id exists
- joined_at exists
- composite primary key exists on (game_id, user_id)
- foreign keys point to games.id and users.id
- ON DELETE CASCADE is present for game_id

### 6. Insert a test game

Inside psql, run:

```sql
INSERT INTO games DEFAULT VALUES
RETURNING *;
```

Verify:

- a row is inserted
- status defaults to waiting
- created_at is populated

### 7. Insert a test game_users row

First, check available users:

```sql
SELECT id, email FROM users;
```

Then insert a row using a valid user id and the game id from Step 6:

```sql
INSERT INTO game_users (game_id, user_id)
VALUES (<game_id>, <user_id>)
RETURNING *;
```

Verify:

- row inserts successfully
- joined_at is populated

### 8. Verify composite primary key behavior

Inside psql, run:

```sql
INSERT INTO game_users (game_id, user_id)
VALUES (<game_id>, <user_id>);
```

Verify:

- insert fails due to duplicate primary key
- error confirms constraint is enforced

### 9. Verify cascade delete behavior

Inside psql, run:

```sql
DELETE FROM games
WHERE id = <game_id>;
```

Then run:

```sql
SELECT * FROM game_users
WHERE game_id = <game_id>;
```

Verify:

- related game_users rows were deleted automatically

### 10. Exit psql

Inside psql, run:

```sql
\q
```

### 11. Final validation (Terminal 2)

Open a **new terminal window** and run:

```bash
npm run check:base:pr
```

Verify:

- build passes
- lint passes
- format check passes
- no errors occur
